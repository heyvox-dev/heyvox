  <name>Task 1: Create BlackHole component integration test file</name>
  <files>tests/test_integration_audio.py</files>
  <action>
Create tests/test_integration_audio.py with the following integration tests. All tests are marked with both `@pytest.mark.integration` (module-level pytestmark) and `@blackhole_installed` so they skip gracefully without BlackHole. These tests do NOT require a running heyvox instance.

Import and reuse helpers from test_e2e.py: `_find_blackhole_device_id`, `generate_tone`, `generate_silence`, `write_wav`. Also import `play_to_blackhole` but note some tests will read from BlackHole directly via pyaudio.

Helper function `_find_blackhole_input_id() -> int | None`: Like `_find_blackhole_device_id` but finds the INPUT device (maxInputChannels > 0). BlackHole 2ch has both input and output on the same virtual device.

Helper function `_record_from_blackhole(duration_secs: float, sample_rate: int = 16000, chunk_size: int = 1280) -> list[np.ndarray]`: Opens a pyaudio stream on the BlackHole input device, records for `duration_secs`, returns list of int16 numpy chunks. This is the core building block — play audio to BlackHole output, record from BlackHole input.

**Test classes:**

1. **TestBlackHoleLoopback** — Sanity check that BlackHole loopback works:
   - `test_tone_round_trip`: Generate a 440Hz tone WAV, play it to BlackHole output (via sounddevice), simultaneously record from BlackHole input (via pyaudio in a thread). Verify the recorded audio has energy above silence threshold (peak > 100). Use threading: start recording thread first, sleep 0.2s, then play. Recording duration = tone duration + 0.5s buffer.
   - `test_silence_round_trip`: Record from BlackHole with nothing playing. Verify peak level is below 50 (digital silence on virtual device should be near-zero).

2. **TestMicProfileCalibrationIntegration** — Real audio through MicProfileManager:
   - `test_calibration_with_blackhole_silence`: Record 2s of silence from BlackHole input, pass chunks to `MicProfileManager.run_calibration()`. Verify noise_floor < 50 and silence_threshold < 175 (3.5 * 50).
   - `test_calibration_with_blackhole_tone`: Play a low-amplitude tone to BlackHole output while recording from input. Pass chunks to `run_calibration()`. Verify noise_floor > 0 and silence_threshold = min(noise_floor * 3.5, 500).
   - `test_calibration_save_and_reload`: Run calibration with BlackHole silence chunks, save via `save_calibration()` to a tmp_path cache dir, create a new MicProfileManager with same cache_dir, verify `get_profile("BlackHole 2ch")` returns the saved values.

3. **TestHeraldStopInterruptIntegration** — Real process killing:
   - `test_stop_kills_real_afplay`: Start a real `afplay` process playing a generated WAV (2s tone at 48000Hz written to tmp file), write its PID to a temp file, patch `HERALD_PLAYING_PID`/`HERALD_QUEUE_DIR`/`TTS_PLAYING_FLAG` to temp paths, call `_cmd_stop()`, verify the afplay process is no longer running (poll() is not None or os.kill(pid, 0) raises).
   - `test_interrupt_kills_real_afplay_preserves_queue`: Same setup but call `_cmd_interrupt()`. Verify afplay is killed AND queue files still exist.

4. **TestCalibrateCommandIntegration** — Real `heyvox calibrate` with BlackHole:
   - `test_calibrate_with_blackhole_device`: Call `_cmd_calibrate` with args `device="BlackHole"`, `duration=2`, `show=False`. Patch `_calibrate_get_cache_dir` to tmp_path but do NOT patch `_calibrate_open_pa` (use real PyAudio). Verify cache file is written with a "blackhole 2ch" entry containing noise_floor and silence_threshold.

5. **TestEchoSuppressionGatingIntegration** — Flag-based coordination:
   - `test_tts_flag_blocks_and_clears_on_stop`: Create TTS_PLAYING_FLAG file (using isolate_flags fixture paths), verify it exists, call `_cmd_stop()` with patched constants pointing to isolate_flags paths, verify TTS_PLAYING_FLAG is removed.
   - `test_recording_flag_written_before_tts_interrupt`: Create RECORDING_FLAG, create TTS_PLAYING_FLAG, call interrupt, verify RECORDING_FLAG still exists (recording should survive interrupt) and TTS_PLAYING_FLAG is removed.

For the afplay tests (TestHeraldStopInterruptIntegration), generate a WAV at 48000Hz sample rate (afplay's preferred rate) using the generate_tone helper with sample_rate=48000. Write to a tempfile. Start afplay as `subprocess.Popen(["afplay", wav_path])`. Give it 0.3s to start before calling stop/interrupt.

All tests should have reasonable timeouts (5s max) and clean up temp files in fixtures.
