**vox/setup/launchd.py** — Service lifecycle management (CLI-01, PROJ-05):

1. `PLIST_DIR = Path.home() / "Library" / "LaunchAgents"`
2. `PLIST_PATH = PLIST_DIR / f"{LAUNCHD_LABEL}.plist"` (import LAUNCHD_LABEL from constants)
3. `GUI_DOMAIN = f"gui/{os.getuid()}"`

4. `write_plist() -> Path`:
   - Generate plist XML using `sys.executable` for ProgramArguments (NOT hardcoded path — research pitfall 3).
   - ProgramArguments: `[sys.executable, "-m", "vox.main"]` (run main module directly, not CLI start, to avoid re-entrant launchctl).
   - RunAtLoad: true, KeepAlive: true.
   - StandardOutPath and StandardErrorPath: `/tmp/vox.log`.
   - EnvironmentVariables: PATH includes `/usr/local/bin:/usr/bin:/bin` plus the directory of sys.executable.
   - Label: `com.vox.listener` from constants.
   - Write to PLIST_PATH, creating PLIST_DIR if needed.
   - Return PLIST_PATH.

5. `bootstrap() -> tuple[bool, str]`:
   - Check if plist exists, if not call write_plist().
   - Run `launchctl bootstrap gui/{uid} {plist_path}`.
   - Return (success, message). If returncode 37 (already loaded), return (True, "Already running").

6. `bootout() -> tuple[bool, str]`:
   - Run `launchctl bootout gui/{uid} {plist_path}`.
   - Return (success, message). If returncode 3 (not loaded), return (True, "Not running").

7. `get_status() -> dict`:
   - Run `launchctl list com.vox.listener`.
   - Parse output: PID (tab) ExitCode (tab) Label.
   - Return `{"loaded": bool, "running": bool, "pid": int|None, "exit_code": int|None}`.
   - If returncode != 0: return loaded=False, running=False.

8. `restart() -> tuple[bool, str]`:
   - bootout() then bootstrap(). Return combined message.

**vox/setup/permissions.py** — macOS permission checks (CLI-03):

1. `PERMISSION_URLS` dict mapping "microphone", "accessibility", "screen_recording" to their `x-apple.systempreferences:` deep-link URLs (from research Pattern 5).

2. `check_accessibility() -> bool`:
   - Import `ApplicationServices` (PyObjC) lazily.
   - Call `ApplicationServices.AXIsProcessTrusted()`.
   - Return bool.

3. `check_microphone() -> bool`:
   - Try to open a brief pyaudio stream (1 chunk read). If it works, mic is accessible.
   - Wrap in try/except — PermissionError or OSError means no permission.
   - Close stream immediately after test.

4. `check_screen_recording() -> bool`:
   - Run `osascript -e 'tell application "System Events" to get name of first window of first process'`.
   - If it returns successfully, Screen Recording is granted. If error, it's denied.
   - This is a heuristic — Screen Recording is needed for text injection via osascript.

5. `open_permission_settings(permission: str) -> None`:
   - `subprocess.run(["open", PERMISSION_URLS[permission]])`.

**vox/setup/wizard.py** — Interactive guided setup (CLI-02, CLI-03, CLI-04):

Use `rich` for all terminal output (Progress, Console, Panel, Spinner).

`run_setup(config: VoxConfig) -> None`:

Step 1: **Welcome banner** — rich Panel with "Vox Setup" title, version.

Step 2: **Permission checks** (CLI-03):
- Check each: accessibility, microphone, screen_recording.
- For each: show green checkmark if OK, red X if missing.
- If missing: print explanation, call `open_permission_settings()`, prompt user to grant and press Enter to re-check.
- Loop re-check up to 3 times per permission.

Step 3: **Model download** (CLI-04):
- Check if Kokoro model is cached: look for `~/.cache/huggingface/hub/models--hexgrad--Kokoro-82M/`.
- If not cached: show rich Progress with spinner, call `huggingface_hub.snapshot_download(repo_id="hexgrad/Kokoro-82M")`.
- Warn about download size (~300MB) and PyTorch dependency.
- If already cached: show green checkmark "Kokoro model already downloaded".

Step 4: **Mic test**:
- Open a 2-second pyaudio recording.
- Show live amplitude level using rich Live display.
- Report max detected level and whether it seems to be working (level > 100 = OK, level == 0 = problem).

Step 5: **Config file**:
- If `~/.config/vox/config.yaml` doesn't exist, offer to create it via `ensure_config_dir()`.
- Show path to config file.

Step 6: **launchd service**:
- Offer to install the launchd plist via `write_plist()`.
- Offer to start the service via `bootstrap()`.

Step 7: **Summary** — show what was set up, what needs attention.

**Important:** Import `rich` lazily inside run_setup() — don't require it at import time. Same for `huggingface_hub`. These are only needed during setup, not normal operation.

**vox/cli.py** — Fill in the stub implementations for service management commands and add setup:

1. `_cmd_start(args)`:
   - Current implementation calls `from vox.main import run; run()`. This is the "direct run" path.
   - Add a `--daemon` flag to the start subparser.
   - If `--daemon`: call `from vox.setup.launchd import bootstrap; success, msg = bootstrap(); print(msg)`.
   - If no `--daemon` (default): keep existing behavior — run the main loop directly (foreground mode, useful for development).

2. `_cmd_stop(args)`:
   - Lazy import `from vox.setup.launchd import bootout`.
   - Call `bootout()`, print result message.

3. `_cmd_restart(args)`:
   - Lazy import `from vox.setup.launchd import restart`.
   - Call `restart()`, print result message.

4. `_cmd_status(args)`:
   - Lazy import `from vox.setup.launchd import get_status`.
   - Call `get_status()`.
   - Print formatted status: "Vox v{version} — {Running (PID {pid}) | Stopped (exit code {code}) | Not installed}".

5. `_cmd_setup(args)`:
   - Lazy import `from vox.config import load_config` and `from vox.setup.wizard import run_setup`.
   - Load config, call `run_setup(config)`.

6. Add `--daemon` flag to `start` subparser: `sub_start.add_argument("--daemon", "-d", action="store_true", help="Start as launchd service (background)")`.

**vox/main.py** — Wire TTS startup and interrupt (TTS-03):

1. At the top of `main()`, after config is loaded and log is initialized:
   - Add: `from vox.audio.tts import start_worker as _start_tts, shutdown as _shutdown_tts, interrupt as _tts_interrupt`.
   - If `config.tts.enabled`: call `_start_tts(config)` to start the TTS background worker.
   - Add to the finally block: if `config.tts.enabled`: call `_shutdown_tts()`.

2. In `start_recording()`:
   - After setting `is_recording = True` and before playing the listening cue:
   - Add: `try: from vox.audio.tts import interrupt as _tts_int; _tts_int(); except ImportError: pass`
   - This immediately pauses any active TTS when the user starts speaking (TTS-03).
   - Use try/except ImportError to gracefully handle case where TTS deps aren't installed.

3. Update the voice command dispatch in `_send_local()`:
   - Currently calls `execute_voice_command(action_key, feedback, tts_script_path=tts_script, log_fn=log)`.
   - Add a check: if `config.tts.enabled` and action_key matches a native TTS command, call the tts module directly:
     - "tts-skip" -> `from vox.audio.tts import skip_current; skip_current()`
     - "tts-stop" -> `from vox.audio.tts import stop_all; stop_all()`
     - "tts-mute" -> `from vox.audio.tts import set_muted, is_muted; set_muted(not is_muted())`
   - Fall through to execute_voice_command() for "tts-next" and "tts-replay" (not yet implemented natively).
   - This means voice commands like "skip" and "stop" now control the native TTS engine when TTS is enabled.

4. Also wire the TTS command file check: In the main loop, between audio read and wake word detection, check for `/tmp/vox-tts-cmd` and dispatch commands. Actually, this is already handled inside the TTS worker thread (Plan 01 Task 2), so no additional wiring needed in main.py for command file IPC.
