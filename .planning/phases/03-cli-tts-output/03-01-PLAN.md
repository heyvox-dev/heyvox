**vox/audio/tts.py** — Complete rewrite. Preserve the existing `check_voice_command()` and `execute_voice_command()` functions (they're used by main.py), but replace everything else and add the TTS engine. The module should contain:

1. **Verbosity enum and filter:**
   - `class Verbosity(str, Enum)` with FULL, SUMMARY, SHORT, SKIP values.
   - `apply_verbosity(text, verbosity) -> str | None` — SKIP returns None, FULL returns text unchanged, SHORT returns first sentence (up to first `.!?`, max 100 chars), SUMMARY truncates at 150 chars at word boundary with "..." suffix.

2. **Volume control (AUDIO-12) and audio ducking (TTS-04):**
   - `_get_system_volume() -> int` — runs `osascript -e "output volume of (get volume settings)"`, returns 0-100. On error returns 50.
   - `_set_system_volume(level: int)` — runs `osascript -e "set volume output volume {level}"`.
   - Volume flow: before playback, read system volume, add `volume_boost` from config (cap at 100), set boosted volume. After playback, restore original in finally block.
   - **Audio ducking (TTS-04):** If `ducking_percent` > 0 and < 100, before playback reduce system volume to `original_volume * ducking_percent / 100`, then set boosted TTS volume. On playback end (finally block), restore original volume. This ensures other system sounds are quieter while TTS speaks. If ducking_percent is 0 or 100, skip ducking (0 = mute others, 100 = no ducking). Store the original volume at the start of each playback item so the finally block always restores correctly.

3. **TTS flag IPC (echo suppression):**
   - `_set_tts_flag(active: bool)` — touch or unlink `TTS_PLAYING_FLAG` from constants. Must be called in try/finally around all playback.

4. **Pipeline (lazy init):**
   - Module-level `_pipeline: KPipeline | None = None`.
   - `_get_pipeline(lang_code='a') -> KPipeline` — lazy-creates on first call. Import `kokoro.KPipeline` inside the function (lazy import pattern, consistent with Phase 1 decisions).

5. **Queue and worker thread:**
   - `_tts_queue: queue.Queue` (unbounded Queue, but enforce MAX_HELD manually before put).
   - `_stop_event: threading.Event` — set by interrupt(), checked between synthesis chunks.
   - `_muted: bool = False` (module-level flag).
   - `_verbosity: Verbosity = Verbosity.FULL` (session-level override, defaults from config on startup).
   - `_worker_thread: threading.Thread | None = None`.

6. **Command file IPC (cross-process control):**
   - `_check_cmd_file()` — reads and deletes `TTS_CMD_FILE` from constants.
   - Parse commands: "skip" -> set stop_event, "stop" -> set stop_event + clear queue, "mute-toggle" -> toggle _muted, "quiet" -> set _verbosity to SHORT.
   - Called at the top of each dequeue iteration in the worker loop and between audio chunks.
   - This gives cross-process CLI control without signals or sockets — simple file-based IPC consistent with the existing flag-file pattern (TTS_PLAYING_FLAG, RECORDING_FLAG).

7. **Public API functions:**
   - `start_worker(config: VoxConfig)` — starts the background worker thread. Reads initial verbosity from `config.tts.verbosity`. Stores ducking_percent from config. Called once at startup.
   - `shutdown()` — sends None sentinel to queue, joins worker thread.
   - `speak(text: str, voice: str = None, speed: float = None, verbosity: str = None)` — Apply verbosity filter (use override if provided, else session-level, else skip). If muted, log and return. Drop oldest if queue >= MAX_HELD. Enqueue (filtered_text, voice, speed). Voice/speed default to config values stored at start_worker time.
   - `interrupt()` — set stop_event + sd.stop(). Called by main.py when wake word or PTT triggers.
   - `skip_current()` — same as interrupt() (stops current, worker picks up next).
   - `stop_all()` — interrupt() + clear the queue.
   - `clear_queue()` — drain the queue without stopping current playback.
   - `set_muted(muted: bool)` — set _muted flag. If muting, also stop_all().
   - `is_muted() -> bool`
   - `set_verbosity(level: str)` — set session-level verbosity.
   - `get_verbosity() -> str`

8. **Worker thread function `_tts_worker(voice_default, speed_default, volume_boost, ducking_percent)`:**
   - Loop: dequeue item, break on None sentinel.
   - Check command file at top of each iteration.
   - Clear stop_event at start of each item.
   - Set TTS flag active.
   - Read system volume. If ducking_percent > 0 and < 100: set system volume to `original * ducking_percent / 100`. Then set TTS playback volume to `original + volume_boost` (capped at 100).
   - Call `_get_pipeline()(text, voice=voice, speed=speed)` — this yields (gs, ps, audio) chunks.
   - For each chunk: check stop_event and command file, if stop set break. Otherwise `sd.play(audio, samplerate=24000)` then `sd.wait()`.
   - In finally: restore original system volume, clear TTS flag, call `_tts_queue.task_done()`.
   - Import `sounddevice as sd` at top of function (lazy import pattern).

**Integration point with main.py** (DO NOT modify main.py in this task — that wiring happens in Plan 02):
- The `interrupt()` function is designed to be called from `start_recording()` in main.py.
- The `check_voice_command()` and `execute_voice_command()` functions remain unchanged — they still dispatch to the bash script. Phase 3 Plan 01 does NOT change how voice commands dispatch; it adds the native TTS engine alongside.

**Anti-patterns to avoid (from research):**
- Do NOT import kokoro or sounddevice at module top level (lazy imports only).
- Do NOT use pyaudio for playback (conflicts with mic input stream).
- Do NOT use sd.play(blocking=True) — use sd.play() + sd.wait() so sd.stop() from another thread interrupts.
- Do NOT create KPipeline per call — single instance, single worker thread.
