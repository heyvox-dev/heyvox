**vox/audio/tts.py** — Complete rewrite. Preserve the existing `check_voice_command()` and `execute_voice_command()` functions (they're used by main.py), but replace everything else and add the TTS engine. The module should contain:

1. **Verbosity enum and filter:**
   - `class Verbosity(str, Enum)` with FULL, SUMMARY, SHORT, SKIP values.
   - `apply_verbosity(text, verbosity) -> str | None` — SKIP returns None, FULL returns text unchanged, SHORT returns first sentence (up to first `.!?`, max 100 chars), SUMMARY truncates at 150 chars at word boundary with "..." suffix.

2. **Volume control (AUDIO-12):**
   - `_get_system_volume() -> int` — runs `osascript -e "output volume of (get volume settings)"`, returns 0-100. On error returns 50.
   - `_set_system_volume(level: int)` — runs `osascript -e "set volume output volume {level}"`.
   - Volume flow: before playback, read system volume, add `volume_boost` from config (cap at 100), set boosted volume, restore original after playback in finally block.

3. **TTS flag IPC (echo suppression):**
   - `_set_tts_flag(active: bool)` — touch or unlink `TTS_PLAYING_FLAG` from constants. Must be called in try/finally around all playback.

4. **Pipeline (lazy init):**
   - Module-level `_pipeline: KPipeline | None = None`.
   - `_get_pipeline(lang_code='a') -> KPipeline` — lazy-creates on first call. Import `kokoro.KPipeline` inside the function (lazy import pattern, consistent with Phase 1 decisions).

5. **Queue and worker thread:**
   - `_tts_queue: queue.Queue` (unbounded Queue, but enforce MAX_HELD manually before put).
   - `_stop_event: threading.Event` — set by interrupt(), checked between synthesis chunks.
   - `_muted: bool = False` (module-level flag).
   - `_verbosity: Verbosity = Verbosity.FULL` (session-level override, defaults from config on startup).
   - `_worker_thread: threading.Thread | None = None`.

6. **Public API functions:**
   - `start_worker(config: VoxConfig)` — starts the background worker thread. Reads initial verbosity from `config.tts.verbosity`. Called once at startup.
   - `shutdown()` — sends None sentinel to queue, joins worker thread.
   - `speak(text: str, voice: str = None, speed: float = None, verbosity: str = None)` — Apply verbosity filter (use override if provided, else session-level, else skip). If muted, log and return. Drop oldest if queue >= MAX_HELD. Enqueue (filtered_text, voice, speed). Voice/speed default to config values stored at start_worker time.
   - `interrupt()` — set stop_event + sd.stop(). Called by main.py when wake word or PTT triggers.
   - `skip_current()` — same as interrupt() (stops current, worker picks up next).
   - `stop_all()` — interrupt() + clear the queue.
   - `clear_queue()` — drain the queue without stopping current playback.
   - `set_muted(muted: bool)` — set _muted flag. If muting, also stop_all().
   - `is_muted() -> bool`
   - `set_verbosity(level: str)` — set session-level verbosity.
   - `get_verbosity() -> str`

7. **Worker thread function `_tts_worker(voice_default, speed_default, volume_boost)`:**
   - Loop: dequeue item, break on None sentinel.
   - Clear stop_event at start of each item.
   - Set TTS flag active.
   - Read and boost volume.
   - Call `_get_pipeline()(text, voice=voice, speed=speed)` — this yields (gs, ps, audio) chunks.
   - For each chunk: check stop_event, if set break. Otherwise `sd.play(audio, samplerate=24000)` then `sd.wait()`.
   - In finally: restore volume, clear TTS flag, call `_tts_queue.task_done()`.
   - Import `sounddevice as sd` at top of function (lazy import pattern).

**Integration point with main.py** (DO NOT modify main.py in this task — that wiring happens in Plan 02 or is deferred to verification):
- The `interrupt()` function is designed to be called from `start_recording()` in main.py.
- The `check_voice_command()` and `execute_voice_command()` functions remain unchanged — they still dispatch to the bash script. Phase 3 Plan 01 does NOT change how voice commands dispatch; it adds the native TTS engine alongside. Voice command dispatch will be updated to call the new TTS functions instead of the bash script in a future task.

**Anti-patterns to avoid (from research):**
- Do NOT import kokoro or sounddevice at module top level (lazy imports only).
- Do NOT use pyaudio for playback (conflicts with mic input stream).
- Do NOT use sd.play(blocking=True) — use sd.play() + sd.wait() so sd.stop() from another thread interrupts.
- Do NOT create KPipeline per call — single instance, single worker thread.

Add four new subcommands to vox/cli.py's `main()` function:

1. **`vox speak <text>`** (CLI-05):
   - Subparser: `speak`, positional arg `text` (nargs="+", join with spaces), optional `--voice` (default from config), optional `--speed` (type=float, default from config), optional `--verbosity` (choices=["full","summary","short","skip"]).
   - Handler `_cmd_speak(args)`:
     - Lazy import `from vox.audio.tts import speak, start_worker, shutdown`.
     - Lazy import `from vox.config import load_config`.
     - Load config, call `start_worker(config)`.
     - Call `speak(text=" ".join(args.text), voice=args.voice, speed=args.speed, verbosity=args.verbosity)`.
     - Import `from vox.audio.tts import _tts_queue` and call `_tts_queue.join()` to wait for playback to finish.
     - Call `shutdown()`.
   - This is a fire-and-forget CLI command — starts TTS, waits for completion, exits. No daemon needed.

2. **`vox skip`** (CLI-06):
   - Handler `_cmd_skip(args)`:
     - Lazy import `from vox.audio.tts import skip_current`.
     - Call `skip_current()`.
     - Print "Skipped current TTS."
   - Note: this only works if a vox process is running (sd.stop() is process-local). For cross-process skip, write a signal file or use SIGUSR2. For v1, document that skip/mute/quiet only work when called within the same process (e.g., via voice commands). For CLI cross-process control, send SIGUSR2 to the running vox process:
     - Read PID from `launchctl list com.vox.listener` output.
     - Send `os.kill(pid, signal.SIGUSR2)` — main.py will need a SIGUSR2 handler (add in Plan 02).
   - Actually, simpler approach for v1: use a **command file** IPC. Write a command to `/tmp/vox-cmd` (e.g., "skip\n"). The main loop or TTS worker checks this file periodically. This avoids signal complexity.
   - **Simplest v1 approach**: Write command to `/tmp/vox-tts-cmd` file. The TTS worker thread checks this file between chunks. Commands: "skip", "stop", "mute", "unmute", "quiet".
   - Add to constants.py: `TTS_CMD_FILE = "/tmp/vox-tts-cmd"`

3. **`vox mute`** (CLI-06):
   - Handler `_cmd_mute(args)`: Write "mute" to TTS_CMD_FILE. Print "TTS muted." (Toggle: if file already contains "mute", write "unmute" instead — actually, just always write "mute-toggle" and let the TTS worker handle toggle logic).
   - Write "mute-toggle\n" to `/tmp/vox-tts-cmd`.

4. **`vox quiet`** (CLI-06):
   - Handler `_cmd_quiet(args)`: Write "quiet\n" to `/tmp/vox-tts-cmd`. Print "TTS verbosity set to short."

**Update the TTS worker** (back in vox/audio/tts.py) to check `/tmp/vox-tts-cmd` between chunks:
- Add `_check_cmd_file()` function that reads and deletes TTS_CMD_FILE.
- Parse commands: "skip" -> set stop_event, "stop" -> set stop_event + clear queue, "mute-toggle" -> toggle _muted, "quiet" -> set _verbosity to SHORT.
- Call `_check_cmd_file()` at the top of each dequeue iteration in the worker loop and between audio chunks.
- Add `TTS_CMD_FILE = "/tmp/vox-tts-cmd"` to vox/constants.py.

This gives cross-process CLI control without signals or sockets — simple file-based IPC consistent with the existing flag-file pattern (TTS_PLAYING_FLAG, RECORDING_FLAG).
