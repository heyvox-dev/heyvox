  <action>
Replace the stub in vox/hud/ipc.py with full HUDServer and HUDClient classes. Keep `SOCKET_PATH = "/tmp/vox-hud.sock"`.

1. **HUDServer class** (used by overlay.py):
   - `__init__(self, path: str, on_message: Callable[[dict], None])` — stores path and callback
   - `start(self)` — launches `_serve` on a daemon thread
   - `_serve(self)` — `os.unlink(path)` (catch FileNotFoundError), bind, listen(1), accept loop spawning `_handle` per connection on daemon threads
   - `_handle(self, conn)` — buffered read with `b"\n"` splitting, `json.loads` each line, call `self._on_message(msg)`. Catch `json.JSONDecodeError` silently. Close conn in `with` block.
   - `shutdown(self)` — sets a `_running = False` flag and unlinks the socket file (best-effort cleanup)

2. **HUDClient class** (used by main.py and mcp/server.py):
   - `__init__(self, path: str = SOCKET_PATH)` — stores path, `self._sock = None`
   - `connect(self)` — creates `socket.AF_UNIX, socket.SOCK_STREAM`, connects. On `FileNotFoundError` or `ConnectionRefusedError`, sets `self._sock = None` (silent degrade)
   - `send(self, msg: dict)` — if `_sock is None`, return. `sendall((json.dumps(msg) + "\n").encode())`. On `BrokenPipeError` or `OSError`, set `self._sock = None`.
   - `close(self)` — close socket if open, set to None
   - `reconnect(self)` — close + connect (for periodic retry)

Use only stdlib: `socket`, `threading`, `json`, `os`. No async.

Replace the entire content of vox/hud/overlay.py with the full HUD overlay process. This is a complete rewrite — the old 18px red dot is replaced by a frosted-glass pill with state machine. Follow patterns from 05-RESEARCH.md precisely.

**Structure overview (top to bottom):**

1. **Module docstring** referencing HUD-01 through HUD-08.

2. **Imports:** `signal`, `sys`, `threading` from stdlib. Lazy AppKit imports inside `main()`:
   - From AppKit: `NSApplication, NSWindow, NSColor, NSView, NSBezierPath, NSTextField, NSButton, NSWindowStyleMaskBorderless, NSScreen, NSBackingStoreBuffered, NSStatusWindowLevel, NSVisualEffectView, NSTextAlignmentCenter, NSAnimationContext`
   - From AppKit (collection behavior constants): `NSWindowCollectionBehaviorCanJoinAllSpaces, NSWindowCollectionBehaviorFullScreenAuxiliary, NSWindowCollectionBehaviorStationary, NSWindowCollectionBehaviorIgnoresCycle`
   - From Foundation: `NSObject, NSTimer, NSMakeRect`
   - Material constant: `try: from AppKit import NSVisualEffectMaterialHUDWindow as HUD_MATERIAL` / `except ImportError: HUD_MATERIAL = 23`

3. **Constants:**
   - `PILL_W_IDLE, PILL_H = 48, 32` (compact idle state — small rounded pill)
   - `PILL_W_ACTIVE = 320` (expanded during listening/processing/speaking)
   - `PILL_MARGIN_TOP = 12` (pixels from top of screen)
   - `ANIM_DURATION = 0.2` (seconds for expand/contract)
   - State colors dict: `{"idle": (0.5, 0.5, 0.5, 0.6), "listening": (1.0, 0.2, 0.2, 0.8), "processing": (1.0, 0.7, 0.0, 0.8), "speaking": (0.2, 0.8, 0.3, 0.8)}`

4. **WaveformView(NSView)** subclass:
   - Instance var `_level = 0.0`, `_num_bars = 5`
   - `setLevel_(self, level)` — stores level, calls `setNeedsDisplay_(True)`
   - `drawRect_(self, rect)` — draws `_num_bars` vertical bars. Center bar is tallest (full rect height * level), outer bars are shorter (scale factor 0.6, 0.4 for outer). Use `NSColor.whiteColor().colorWithAlphaComponent_(0.9)` and `NSBezierPath.fillRect_` for each bar. Bar width = `rect.size.width / (num_bars * 2 - 1)`, spacing = same as bar width.

5. **HUDContentView(NSView)** subclass (Pattern 4 from research):
   - `hitTest_(self, point)` — delegates to super; if hit is self, return None (click-through background); otherwise return the hit subview (buttons respond)
   - This enables mixed click-through / clickable regions.

6. **State machine function `_apply_state(state_str, window, content_view, waveform_view, transcript_label, tts_controls, color_overlay)`:**
   - Updates color overlay background based on state string
   - If state is "idle": animate pill to PILL_W_IDLE, hide waveform/transcript/tts_controls, `window.setIgnoresMouseEvents_(True)`
   - If state is "listening": animate pill to PILL_W_ACTIVE, show waveform, hide transcript/tts_controls, set color to red
   - If state is "processing": keep PILL_W_ACTIVE, hide waveform, show transcript label ("Transcribing..."), set color to amber
   - If state is "speaking": keep PILL_W_ACTIVE, hide waveform, show tts_controls, set color to green, `window.setIgnoresMouseEvents_(False)`
   - Pill animation uses `NSAnimationContext.beginGrouping()` / `setDuration_(ANIM_DURATION)` / `window.animator().setFrame_display_(new_frame, True)` / `endGrouping()`. Reposition x to keep centered on screen.
   - Also update `content_view` frame and subview frames to match new window size.

7. **NSObject dispatcher** for thread-safe UI updates (Pattern 5 from research):
   - `_Dispatcher(NSObject)` with an `applyMessage_(self, msg_dict)` method
   - The method checks `msg_dict["type"]`:
     - `"state"` → call `_apply_state(msg_dict["state"], ...)`
     - `"audio_level"` → call `waveform_view.setLevel_(msg_dict["level"])`
     - `"transcript"` → call `transcript_label.setStringValue_(msg_dict["text"])`
     - `"tts_start"` → apply speaking state, optionally show text snippet
     - `"tts_end"` → apply idle state
     - `"queue_update"` → (v1: ignore, future: show badge)
     - `"error"` → (v1: log to stderr)
   - Store references to window/views as instance vars set during init

8. **TTS control buttons** (created during main, added to content_view):
   - Two `NSButton` instances: "Skip" and "Stop"
   - On click, they write to `/tmp/vox-tts-cmd` (same IPC as CLI commands from Phase 3):
     - Skip button writes "skip"
     - Stop button writes "stop"
   - Style: bezelless, white text, small font (11pt), positioned at right edge of pill
   - Use `setTarget_` / `setAction_` with an NSObject action handler

9. **`main()` function:**
   - `app = NSApplication.sharedApplication()`, `app.setActivationPolicy_(2)` (no dock icon)
   - Get screen frame from `NSScreen.mainScreen().frame()`
   - Create `NSWindow` with borderless style, `NSStatusWindowLevel + 1`, clear background, opaque=False
   - Collection behavior: `canJoinAllSpaces | fullScreenAuxiliary | stationary | ignoresCycle` (HUD-07)
   - Create `NSVisualEffectView` as content backing: `setMaterial_(HUD_MATERIAL)`, `setBlendingMode_(0)`, `setState_(1)`, layer-backed with `cornerRadius = PILL_H / 2`, `masksToBounds = True` (HUD-06)
   - Create `HUDContentView` overlay on top of the visual effect view for hit-testing
   - Create a color overlay view (simple NSView with background color) for state coloring, with rounded corners matching pill, alpha 0.3
   - Create `WaveformView` as subview (HUD-02), initially hidden
   - Create `NSTextField` for transcript (HUD-03): non-editable, no background, white text, center-aligned, initially hidden
   - Create TTS control buttons (HUD-04), initially hidden
   - Set initial window frame: centered horizontally, PILL_W_IDLE x PILL_H, positioned `PILL_MARGIN_TOP` below screen top
   - `window.setIgnoresMouseEvents_(True)` (idle = click-through)
   - `window.orderFrontRegardless()`
   - Create `_Dispatcher` instance with references to all views
   - Start `HUDServer` on daemon thread with callback that dispatches via `_dispatcher.performSelectorOnMainThread_withObject_waitUntilDone_("applyMessage:", msg, False)`

   **IMPORTANT dispatch note:** `performSelectorOnMainThread_withObject_waitUntilDone_` passes an ObjC object. Python dicts bridge to NSDictionary automatically via PyObjC, so this should work. If it doesn't (test!), fall back to serializing to JSON string and deserializing in `applyMessage_`.

   - Install SIGTERM/SIGINT handlers using the existing NSTimer pattern from the current overlay.py (proven pattern)
   - Clean up socket file in a finally or atexit handler
   - `app.run()`

10. **`if __name__ == "__main__": main()`**

**Key implementation notes:**
- All AppKit calls must happen on the main thread. The HUDServer callback runs on a background thread — it MUST dispatch to main thread via `performSelectorOnMainThread` or NSTimer.
- Use `NSScreen.mainScreen()` every time window position needs recalculating (don't cache).
- The color overlay should be semi-transparent (alpha 0.3) so the frosted glass shows through.
- Waveform bars should use white color at 0.9 alpha for visibility on the dark HUD material.
- For TTS buttons, use `TTS_CMD_FILE` from `vox.constants` for the file path. Import it inside the button action handler to avoid top-level vox imports that could fail when running overlay.py as a standalone script.
